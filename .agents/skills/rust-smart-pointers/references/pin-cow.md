# Pin and Cow

## Pin<P> In Depth

### The Problem: Self-Referential Types

```rust
struct SelfRef {
    data: String,
    ptr: *const String,  // points to `data` field
}
// If SelfRef is moved, `ptr` becomes dangling!
```

### How Pin Solves It

`Pin<P>` wraps a pointer `P` and guarantees the pointee won't move:

```rust
use std::pin::Pin;

// Pin<Box<T>> — value is heap-allocated and can't be moved
let pinned: Pin<Box<SelfRef>> = Box::pin(SelfRef::new());

// Pin<&mut T> — borrowed value guaranteed not to move
// Only constructible if T: Unpin (safe to move) OR via unsafe
```

### Unpin Trait

- `T: Unpin` — safe to move even when pinned (most types)
- `T: !Unpin` — must not be moved once pinned (futures, self-referential types)

```rust
// Most types are Unpin (compiler auto-implements)
struct Normal { x: i32 }  // Normal: Unpin

// Opt out with PhantomPinned
use std::marker::PhantomPinned;
struct NotMovable {
    data: String,
    _pin: PhantomPinned,  // makes this !Unpin
}
```

### Pinning in Async

Futures generated by `async fn` are `!Unpin` because they may be self-referential:

```rust
async fn example() {
    let data = vec![1, 2, 3];
    some_async_fn(&data).await;  // &data stored in future's state
    // If future moved between awaits, &data would dangle
}
```

That's why executors pin futures before polling:

```rust
// Executor does this internally:
let future = Box::pin(my_async_fn());
future.as_mut().poll(cx);
```

### pin! Macro (Stack Pinning)

```rust
use std::pin::pin;

let mut future = pin!(async { 42 });
// `future` is Pin<&mut impl Future<Output = i32>>
// Cannot be moved after this point
```

### Pin Projections

Access fields of a pinned struct:

```rust
use pin_project::pin_project;

#[pin_project]
struct MyFuture {
    #[pin]
    inner: InnerFuture,  // pinned field — stays pinned
    counter: u32,        // unpinned field — can be moved/mutated freely
}

impl Future for MyFuture {
    type Output = ();
    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<()> {
        let this = self.project();
        // this.inner: Pin<&mut InnerFuture>
        // this.counter: &mut u32
        this.inner.poll(cx)
    }
}
```

## Cow<'a, B> In Depth

### Basics

```rust
use std::borrow::Cow;

enum Cow<'a, B: ToOwned + ?Sized> {
    Borrowed(&'a B),
    Owned(<B as ToOwned>::Owned),
}
```

Common instantiations:
- `Cow<'_, str>` — either `&str` or `String`
- `Cow<'_, [u8]>` — either `&[u8]` or `Vec<u8>`
- `Cow<'_, Path>` — either `&Path` or `PathBuf`

### Use Cases

#### Conditional transformation

```rust
fn normalize(input: &str) -> Cow<'_, str> {
    if input.contains('\t') {
        Cow::Owned(input.replace('\t', "    "))
    } else {
        Cow::Borrowed(input)  // no allocation needed
    }
}
```

#### Return either borrowed or owned

```rust
fn get_name(user: &User) -> Cow<'_, str> {
    if let Some(ref name) = user.display_name {
        Cow::Borrowed(name)
    } else {
        Cow::Owned(format!("User #{}", user.id))
    }
}
```

#### Reduce allocations in batch processing

```rust
fn process_lines(lines: &[&str]) -> Vec<Cow<'_, str>> {
    lines.iter().map(|line| {
        if line.starts_with('#') {
            Cow::Borrowed(&line[1..])  // just slice — no allocation
        } else {
            Cow::Owned(line.to_uppercase())  // allocate only when needed
        }
    }).collect()
}
```

### Cow Behavior

```rust
let cow: Cow<'_, str> = Cow::Borrowed("hello");

// Deref to &str (free)
let s: &str = &cow;

// Convert to owned (clones only if borrowed)
let owned: String = cow.into_owned();

// Mutate (clones only if borrowed)
let mut cow = Cow::Borrowed("hello");
cow.to_mut().push_str(" world");  // now Owned
```

### Performance Characteristics

| Operation | Borrowed | Owned |
|-----------|----------|-------|
| Creation | Free (pointer copy) | Allocation |
| `.deref()` | Free | Free |
| `.to_mut()` | Clones → Owned | Returns `&mut` |
| `.into_owned()` | Clones | Free (moves out) |
| Drop | Free | Deallocates |
